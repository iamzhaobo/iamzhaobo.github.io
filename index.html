认识URLSearchParams API

发表于 2017-09-06   |   890 字   |   4 min | 分类于 JavaScript

网页URL编码是key1=value1&key2=value2的形式，无论是对URL的解析还是表单序列化后，均是以这种方式进行展现，要用到里面的参数，我们通常会把这个字符串转化成更加直观的形式(如对象),以前我们都是通过自己写一些方法来实现这个过程，而现今，我们有了URLSearchParams API，就可以方便的对URL编码进行解析与其他相关的操作。

传统的实现方法

这里以网页的URL为例，假如这里的URL=’http://127.0.0.1:8080/test.html?name=zhaobo&age=5'，我们想把参数列表转化为更加直观的对象的形式，通过下面的代码即可实现。

var par = {};
var params = location.search.slice(1);
var entries = params.split('&');
entries.forEach(function(o,i) {
    var arr = o.split('=');
    var key = arr[0];
    var val = arr[1];
    par[key] = val;
})
// {name: 'zhaobo',age: 5}
console.log(par)

这里使用BOM里面location.search得到URL地址中的参数，但是它包含了’?’这个字符，所有我们将第一个字符除去，得到的即是有用的参数列表字符串，后面再根据’&’分割成数组后，在对数组中的元素根据中间的’=’再次进行分隔，最后汇总成一个我们需要的对象。

这样得到这个参数列表的各值就非常的简单了。但是，要进行一些其他的操作，如设置参数、指定参数、追加等又需要不少的代码才能实现。为此，我们可以使用URLSearchParams。

URLSearchParams

基本使用

URLSearchParams提供了对参数操作的各种方法，它是一个构造函数，所以，使用它的第一步就是要new URLSearchParams(param)，构造函数内部有一个参数，一般为字符串，即是我们需要操作的参数列表字符串，构造函数会返回一个对象，所有对参数的操作操作都是通过这个对象来完成的。

它有以下这些基本的方法；

var usp = new URLSearchParams(params);

usp.append(k,v) // 追加一个新的key/value
usp.delete(k> // 删除一个参数项
usp.get(k) // 根据key获取对应value的值
usp.getAll(k) // 获取这个key的所有value，返回其组成的数据
usp.has(key) // 判断列表中是否有某个key
usp.sort() // 按key排序(执行在地址栏排序，不返回任何值)
usp.toString() // 返回参数组成的字符串

其余都是些简单易理解的API，具体示例可以看看这里

遍历方法

URLSearchParams生成的对象有三个方法均支持Iterator的迭代,可以直接用for-of循环。它们包括一下方法。

usp.keys() // 参数中所有的key
usp.values() // 参数中所有的value
usp.entries() // 参数中所有的[key,value]

for (let i of s.keys()) {
    // name age
    console.log(i);
}

for (let i of s.values()) {
    // zhaobo 5
    console.log(i)
}

for (let i of s.entries()) {
    //  ["name", "zhaobo"]
    // ["age", "5"]
    console.log(i)
}

for (let i of s) {
    //  ["name", "zhaobo"]
    // ["age", "5"]
    console.log(i)
}

值得一提的是，这里遍历s和s.entries()的结果是一样的，所以一般就遍历s对象即可。

扩展(结合pushState\URL)

结合history中的相关API，再利用URLSearchParams,就可以在改变参数的同时改变浏览器地址栏显示的效果。

var s = new URLSearchParams(params);
s.set('name','jiaqiY');
history.replaceState(null,'',location.pathname + '?' + s.toString())；

这里，利用URLSearchParams改变name的值，然后利用history.replaceState方法，就可以达到上述的效果。

另外，URLSearchParams还可以与URL接口结合使用。

var url = new URL(location);
var foo = url.searchParams.get('name') || 'somedefault';

你不知道的padding

发表于 2017-08-27   |   1,031 字   |   4 min | 分类于 CSS

最近在二刷张鑫旭老师的《深入理解CSS系列》，以下内容均是课程笔记与实践，更多详细的内容请看这里.

for block

padding是盒模型的重要组成部分，对于块级元素来说，又可以分为两类。一是W3C标准盒模型，即在box-sizing: content-box的情况下，padding的增加后会导致整个元素尺寸的增大；二是IE盒模型，即在box-sizing: border-box的情况下，padding的增加不会导致元素尺寸的增加，增加的padding会导致content区域减少。

如上是知乎的主页，利用这一特性，在对盒子进行绘制的时候，一般可以采用IE的盒模型，然后利用padding将四周的留白写出来。

虽然在border-box的情况下，padding不会对块级元素的尺寸造成影响，但是当水平或垂直方向上的padding之和大于width或height之后，盒子就会“爆炸”了，如这里的例子，当发生了这种情况时，content区域将会变成0，width或height将只有padding决定。

for inline

在inline下，padding有一个“奇怪”的特征。

三个暖心却鲜为人知的CSS属性

平时工作常用到的CSS属性就那几十个，在编写代码中用到的也不是特别深的东西，不过对于一些生僻的属性，使用了之后可以大大的方便我们的开发工作。

pointer-events

在不考虑SVG元素的情况下，这个属性对普通的元素有两个值：auto(默认值)和none；在将值设置为none的情况下，有两个含义，一是此元素不能不能捕获任何的鼠标事件，而是鼠标会完全忽略测元素，穿够作用于下一层元素。

看结构代码：

    <script>
        var canvas = document.querySelector('#c');

        context.fillRect(0,0,500,400);

        var btnWrap = document.querySelector('.btn-wrap');
        btnWrap.addEventListener('click',function() {
            alert('Btn Wrap');
        })

        var btn = document.querySelector('button');
        btn.addEventListener('click',function(event) {
            alert('Fuck')
        })
    </script>
</body>

user-select

这一个属性也是与选中文本相关的，在网上有些资料为了保护版权，有些就会采用这样一种(比较low)的方法，它有四个属性值none、text、all、element；

text是它的默认值，表示可以选择文本；
none则是需要用到这个属性时最为常用的，表示不能选中文本；
all是指能够在单击的时候一并选中当前文字，就好似我们双击选中一个英文单词一样；
element表示可以选择文本，但选择范围受元素边界的约束；

Python函数的几种参数类型

发表于 2017-08-09   |   2,172 字   |   8 min | 分类于 Python

以下代码均以Python3为基础理解。

初识Python函数

大部分常见的语言如C、Java、PHP、C#、JavaScript等属于C系语言，Python不属于他们中的一员（ruby亦然）。在这些语言中，Python也属于比较新奇的一派，就函数来说，它没有大括号，用def关键字定义一个函数，定义后用:然后换行tab指定函数函数的范围，当然也不存在什么分号。

作为一个函数，那个它肯定是有参数的，Python对于参数的定义比较繁杂，如果了解ES6，那么在Python函数的各个参数中差不多都能够找到对应的ES6定义，因为JS是我最为熟悉的编程语言，所以在下面介绍各种参数类型时都会对照JS中对应的类型，以加强理解。

Python函数参数分为位置参数、默认参数、可变参数、关键字参数、命名关键字参数五大类。

位置参数

这是最大众化的一种参数类型了，就是函数中定义确定的形参，然后调用函数的实参传入一一对应的参数，参数的位置、个数都不得有偏差。不同于JS中实参与形参可不同个数，这里的要求是非常严格的。

def desc(name,age,city)
    print('name = ' , name,'age = ' , age,'city = ' , city)

# name =  zhaobo age =  12 city =  luzhou
desc('zhaobo',12,'luzhou')

默认参数

ES6中也有默认参数这种用法，Python中的用法与在ES中的用法出差不多的。在定义形参时即指定好默认参数，如果在实参没有传入这个位置的参数，那么这里的参数就为默认参数，默认参数要写在位置参数后面。

def desc(name,gender,city = 'luzhou',age = 15):
print('name = ' , name,'gender = ' , gender,'city = ' , city,'age = ' , age)

desc('zhaobo','male') # name =  zhaobo gender =  male city =  luzhou age =  15
desc('YeJ.Q','female','Deyang') # name =  YeJ.Q gender =  female city =  Deyang age =  15
desc('WeiC.M','female',age = 16) # name =  WeiC.M gender =  female city =  luzhou age =  16

如上代码，desc函数有四个参数，其中后面两个是默认参数，第一种调用方式只传了必选的两个位置参数，那么通过打印结果来看，name和age是我们传进去的值，后面的city和age是默认参数中的值；第二种方式按照顺序三个值，前两个必选参数自然不必说，第三个参数覆盖了默认参数的值，第四个参数还是默认参数的值，是实际写代码的时候，要将变化小的参数写在参数列后面；最后一种调用方式，我们想的是在city上取默认值，但是在age上需要传新值，这个就在实参里面直接指定age的值，所以体现出默认参数的传递不一定要按照参数的顺序来传值。

总结：

  1. 默认参数必须在必选参数之后；
  2. 当函数有多个参数时，把变化大的参数放前面，变化小的参数放后面。变化小的参数就可以作为默认参数。
  3. 默认参数也可以不安默认的顺序提供参数，enroll(‘Adam’, ‘M’, city=’Tianjin’)，这样调用即可
  4. 默认参数必须指向不变对象（str、None、int、float等）！不可指向list

可变参数

ES6中新增了一种rest参数（不定参数），是JS中arguments的语法糖，在传参的时候可传不定个数的参数，在函数中打印出不定参数是一个数组。

在Python中也有类似的语法，叫做不定参数，他的参数用*param定义，写在函数的最后（有且只能写一个），进行传参的时候，也可传也可不传参数，参数的个数也可变。

def power(name,*desc):
# 打印出来是一个tuple
# ('12', 'male', 'student')
print(desc) 

power('zhaobo','12','male','student') 
power('zhaobo',*['12','male','student'])
power('zhaobo',*('12','male','student'))

上述代码定义了使用不定参数的三种类型

  1. 完全体现“不定”两个字的含义，在对位置参数进行了对应的传参之后，剩下的就是不定参数，注意这里总共传入了四个参数，第一个是位置参数，后三个都将作为不定参数传入，在函数中作为不定参数打印出来的是一个元组(tuple)；
  2. 传入了两个参数，第一个是位置参数，第二个在列表(list)的前面加上了一个*，这样就可以直接传入一个列表，然后打印结果与第一种是一样的。利用这一个语法糖就可以直接将参数传入一个列表。
  3. 这种方式与传入列表几乎无异，只是可以传入一个元组。

关键字参数

在使用JS尤其是jQuery的时候，常见用对象当作参数传递，在Python中也有类似的用法，而且功能也非常强大。

def foo(name,**kw):
# {'age': 8, 'gender': 'male'}
print(kw)

foo('zhaobo',**{'age': 8,'gender': 'male'})
foo('zhaobo',age = 8,gender = 'male')

关键字参数有两个调用方式，第一种，最简单直接的调用，可以在字典前加入**，然后直接传入字典，打印出来之后就是传入的这一个字典；第二种，将字典的key-value写成对应的表达式，然后每一个表达式对应一个参数传入，最后也会打印出这个字典，注意在写参数的时候，key不能用引号包起来。

命名关键字参数

前面的关键字参数是传一个字典到函数中，然后在函数中打印出来也是一个字典的形式。命名关键字参数，即是指定这个字典的key，使得传入的参数只能是当前指定的这些key。如下代码：

# 一般情况
def power(name,*,age,gender):
    print(name) # zhaobo
    print(age) # 12
    print(gender) # male

power('zhaobo',age = 12, gender = 'male')
power('zhaobo',**{'age': 12,'gender': 'male'})

# 有可变参数的情况
def power(name,*kw,age = 10,gender):
    print(name) # zhaobo
    print(kw) # ('param1', 'param2', 'param3')
    print(age) # 12
    print(gender) # male

power('zhaobo','param1','param2','param3',age = 12,gender = 'male')
power('zhaobo','param1','param2','param3',**{'age': 12, 'gender': 'male'})
power('zhaobo','param1','param2','param3',**{'gender': 'male'})

上述代码展示了对命名关键字调用的两种形式。在一般的形式中，命名关键字参数用*分隔，注意*两边要打上逗号，然后就是指定命名命名关键字参数的key值，在调用函数时，有两种方式，一是key = value的形式，key与定义函数的命名关键字参数对应(顺序可以不一样)，注意这里的key不要用引号包起来；另一中方式就是直接传一个字典，然后在字典前面加上**。

在有可变参数的情况下，对函数的定义就不用再单独写*了，指定在可变参数后面定义好关键字参数即可，在调用上，也是有key = value，直接传字典两种形式，在可变参数后面指定关键字参数即可。

需要注意的是，命名关键字参数也可以指定默认值，指定了默认值即代表这个key可传可不传，如果没有指定默认值那个key的参数多传或者是漏传都会导致语法错误。

多种参数类型

在Python中定义函数，上述的五种参数是可以混合定义的，参数定义的顺序必须是：必选参数、默认参数、可变参数、命名关键字参数和关键字参数。

# 定义一个函数，包含上述若干种参数：
def f1(a, b, c=0, *args, **kw):
    print('a =', a, 'b =', b, 'c =', c, 'args =', args, 'kw =', kw)

def f2(a, b, c=0, *, d, **kw):
    print('a =', a, 'b =', b, 'c =', c, 'd =', d, 'kw =', kw)


# 在函数调用的时候，Python解释器自动按照参数位置和参数名把对应的参数传进去。
>>> f1(1, 2)
a = 1 b = 2 c = 0 args = () kw = {}
>>> f1(1, 2, c=3)
a = 1 b = 2 c = 3 args = () kw = {}
>>> f1(1, 2, 3, 'a', 'b')
a = 1 b = 2 c = 3 args = ('a', 'b') kw = {}
>>> f1(1, 2, 3, 'a', 'b', x=99)
a = 1 b = 2 c = 3 args = ('a', 'b') kw = {'x': 99}
>>> f2(1, 2, d=99, ext=None)
a = 1 b = 2 c = 0 d = 99 kw = {'ext': None}

# 最神奇的是通过一个tuple和dict，你也可以调用上述函数：
>>> args = (1, 2, 3, 4)
>>> kw = {'d': 99, 'x': '#'}
>>> f1(*args, **kw)
a = 1 b = 2 c = 3 args = (4,) kw = {'d': 99, 'x': '#'}
>>> args = (1, 2, 3)
>>> kw = {'d': 88, 'x': '#'}
>>> f2(*args, **kw)
a = 1 b = 2 c = 3 d = 88 kw = {'x': '#'}

所以，对于任意函数，都可以通过类似func(args, *kw)的形式调用它，无论它的参数是如何定义的。

JSON.stringify的一些罕见的使用方法。

关于JSON.stringify

作为一名前端开发人员，从接触到Ajax开始，我们边接触到了JSON.stringify与JSON.parse这两个函数。JSON.parse通常用于将服务器传过来的JSON格式的字符串转化成JSON对象或数组，而JSON.stringify则正好相反，它将一个JSON值转化成了JSON字符串。

函数的使用

基本使用

const data = {
    name: 'Guanruyi',
    desc: 'He pick up soap',
    partner: undefined,
    girlfriend: null,
    do: function() {
        console.log('papapa');
    }
}

console.log(JSON.stringify(data))
// {"name":"Guanruyi","desc":"He pick up soap","girlfriend":null}

这里的常量data是一个JavaScript值，下面利用JSON.stringify将之转化为一个字符串，但是，这里的partner与do属性不能转化出字符串来，它们对应的值分别是undefined和一个函数。

将undefined和函数体作为字符串显示，这里就引出了JSON.stringify的后面两个可选参数JSON.stringify(value[,replacer[,space]])

replacer参数

const data = {
    name: 'Guanruyi',
    desc: 'He pick up soap',
    partner: undefined,
    girlfriend: null,
    do: function() {
        console.log('papapa');

const replacerFn = function(k ,v) {
    if(typeof v === 'function') {

    }
    if (v === undefined) {

    }
    return v
}

console.log(JSON.stringify(data, replacerFn))
// {"name":"Guanruyi","desc":"He pick up soap","partner":"undefined","girlfriend":null,"do":"function () {\n\t\t\t\tconsole.log('papapa');\n\t\t\t}"}

replacer是一个回调函数，有两个参数，key和value，分别对应当前对象的键值，除了想上述代码一个对它的function与undefined返回特定值之外，与map函数类似，你可以根据key与value的值对它进行任何操作，返回你想要的值，故叫replacer。

浅析JavaScript数组的9种迭代方式

发表于 2017-07-15   |   2,263 字   |   9 min | 分类于 JavaScript

许多初学者乃至有一定前端编程经验的人往往也道不清JavaScript中数组的几种遍历方式，因为除了常用的几种for之外，对与另外的几种回调方法，在日常编码中使用的也比较少，这里写几个Demo，从易到难，把JavaScript中涉及到的几种数组遍历方法梳理一下。

简介

JavaScript数组的遍历方法按照语法规则来分，我将它们分为两类，一类是以for、for-in、for-of为代表的for循环，另外一类则是以forEach、map、filter等为代表的回调型遍历方法。对应前者，我们是使用的最多的，然后对于后面的几种遍历方法，可能大多数人对于他具体的运行流程，返回值等都不是很清楚。

for型

普通for循环

最简单、最典型、最常用、同时也是效率最高的for循环

var arr = ['S','N','Q','U'];
for (let i = 0,len = arr.length; i < len; i++) {
    console.log(arr[i]); // 依次输出‘S’，‘N’，‘Q’，‘U’；
}

这个方式有两点需要注意的地方，第一，在ES6上，因为块作用域的原因，for循环要使用let声明变量而非var，第二，对于遍历时需要的是固定length的数据，那么就应像上述代码一样大len与i一同声明，或者将之放在for之前，因为如同for(let i = 0; i < arr.length; i++){}这样的代码，它的执行顺序是首先执行let i = 0(且只会执行一次)，然后判断i < arr.length是否成立，接着执行{}内的代码块，最后i++，第二次在从i < arr.length开始执行，这样的话，每一次循环都会重新计算arr的length，相对来说消耗了更多的性能。

for-in

这一种循环方式最适合对对象进行遍历，同时也可以对数组进行遍历，不过它的效率也不高，仅仅是普通for循环的1/7。

首先看它对对象的遍历

let dude = {
    name: 'Denglei',
    age: 22,
    desc: function() {
        console.log(`I'm ${this.name}, and ${this.age} years old.`);
    }
}
for (let key in dude) {
    // 依次输出dude这个对象的键值对，
    // 其中desc的value以字符串的形式打印整个函数体
    console.log(key,dude[key])
    if (typeof dude[key] === 'function') {
        // I'm Denglei, and 22 years old.
        dude[key]();
    }
}

以上述代码为例，for (let key in dude)，dude是我们制定的一个对象，而key则是这个对象的全部的key，在循环体中通过dude[key]的方式取出对象的value。

for-in遍历数组

var arr = ['S','N','Q','U'];
arr.name = 'PHP is the best programming language';
Array.prototype.proAttr = 'prototype attrbute';
for (let i in arr) {
    console.log(i,arr[i])
    console.log(typeof i)
}

运行结果

在只针对var arr = ['S','N','Q','U']这个数组的情况下，for-in数组得出的结果与遍历对象是非常类似的，i代表数组的所以，arr[i]则代码数组的值；但是，奇怪的事情发生了，在这里，i的类型居然是字符串，数组的索引怎么可能是字符串呢？但是，在JavaScript中，确实它的内部实现就是这样，数组在内存中不是存在连续的内存中，数组的索引并不是它在内存中的偏移量，但同时，我们平时是通过arr[i]（这里的i当然是一个Number），取出数组的，这又是怎么回事呢？因为JavaScript已经把i强制转换成了String类型的了，所以我们就感觉是在操作一个“真正的数组”一样。

然后这里for-in把name与原型中的proAttr也遍历了出来，这就是for-in对数组不友好的地方了，因为涉及到要去原型找东西，故它的性能就这么低了，但同时，Array的length属性没有遍历出来的原因是length这个属性的特性里面enumerable是false的，当然不能遍历出来了（具体可以看我之前关于对象属性的文章）。正是因为这个原因，ES6推出了for-of。

总结就是for-in非常适用于对象，不要用在数组上。

for-of

针对for-in对遍历数组的种种不便，ES6推出了for-of循环方式。

var arr = ['S','N','Q','U'];
arr.name = 'PHP is the best programming language';
Array.prototype.proAttr = 'prototype attrbute';
for (let item of arr) {
    console.log(item) // S N Q U
}

没有那么多乱七八糟的东西，遍历出来的就是“纯数组”里面的几个元素，不会去属性和原型里面找东西说明了它的性能肯定不会比for-in差。

另外，下面即将提到的forEach，它不支持break，continue，return，这些，for-of通通支持；for-of不仅支持数组，它也支持ES6中一切有Iterator属性的对象，如nodeList等；同样它也支持对字符串进行遍历，即将字符串看作Unicode来进行遍历，还有Map与Set也可以用for-of。

最后，普通对象还是交给for-in，不要使用for-of。

回调型

既然将它们归为回调型，则它们共同的特定就是均要写成高阶函数传函数参的形式，这个回调的函数里有两个值(item,index)（除了reduce），item是当前遍历的那一个值，index则是它的索引（Number型）。

上面说的是它们的相同点，对于它们的功能，返回值等都各有差异，下面通过代码进行区分。

forEach

let arr = ['I','am','so','busy','recently'];
arr.name = 'LP';
Array.prototype.status = 'Dead';
var result = arr.forEach(function(item,i) {
    console.log('item',item);
    console.log('i',i)
})
console.log('result',result)

功能大而全的for循环，后面五中遍历方式均可以通过这个方式实现。这里需要注意的是，与for-in对数组循环不一样的是，这里循环出来的索引是Number型，而非字符串型的。另外原型与附加的其他属性也是不能被遍历出来的，所以forEach就适用于这种纯纯的数组。同理后面的几种遍历方式也是这样的，只能处理“纯数组”。

map

let arr = ['I','am','so','busy','recently'];
let result = arr.map(function(item,i) {
    return item + '!';
})
console.log(result);// ["I!", "am!", "so!", "busy!", "recently!"]

map的作用是处理数组的每一个元素，然后再返回一个新的数组。这个新数组就是map方法的返回值，
而这个新数组的元素又是回调函数内的每一个返回值决定的。简而言之就是通过某种计算，让一个数组返回一个新的数组。

filter

let arr = ['I','am','so','busy','recently'];
let result = arr.filter(function(item,i) {
    return item.length > 1
})
console.log(result);// ["am", "so", "busy", "recently"]

filter同样是返回一个新的数组，与map不同的是，filter新数组的元素还是由之前的那些元素构成的，不过经过了一次筛选。

reduce

let arr = ['I','am','so','busy','recently'];
let result = arr.reduce(function(prev,next) {
    return prev + next;
})
console.log(result);// Iamsobusyrecently

让数组中的前项与后项进行某种计算，最终算出最终值。如进行累加等操作。

some

let arr = ['I','am','so','busy','recently'];
let result = arr.some(function(item,index) {
    return item.length > 2;
})
console.log(result);// true

some方法返回一个boolean值，bool值由回调中的返回值决定，只要回调中的retrun了一次或以上的true，则some方法返回true。应用的数组上，就是只要数组某一元素满足一个条件，则最终some方法返回true。

every

let arr = ['I','am','so','busy','recently'];
let arr = ['I','am','so','busy','recently'];
let result = arr.every(function(item,index) {
    return item.length > 2;
})
console.log(result);// false

与some方法类似，不过这里需要回调函数内每一次都返回true，即数组的每一个元素都要满足自定义条件。

jQuery中的each

最后，作为补充，简单介绍一下jQuery中的each，这和forEach非常的相似，在使用上最大的区别获取就是回调参数顺序是相反的。

它有两种形式，一是直接对一数组调用each方法，然后传入回调函数，二是$.each(arr/obj，i,j)两个形式。

let arr = ['I','am','so','busy','recently'];
let obj = {
    name: 'Java',
    desc: 'Java + You'
}

// 这里需要将arr转换成jQuery对象，不然会保存
$(arr).each(function(index,item) {
    // 分别打印出数组的索引与元素值
    console.log(index,item) 
})

// 对对象的遍历没有意义
$(obj).each(function(key,val) {
    // 0,{name: "Java", desc: "Java + You"}
    console.log(key,val);
})

// 报错：obj.each is not a function 
obj.each(function(key,val) {
    console.log(key,val)
})

// arr转不转化成jQuery对象都一样
$.each(arr,function(key,val) {
    // 分别打印出数组的索引与元素值
    console.log(key,val);
})

// obj不能写成$(obj),否则将打印 0,{name: 'Java',desc: 'Java + You'}
$.each(obj,function(key,val) {
    // 打印obj的键值对
    console.log(key,val);
})

jQuery的each方法适用与数组，要注意的是需把数组转化成jQuery对象；$.each适用与数组与对象，且不用转化成jQuery对象，$.each显然更加的强大，有点for-in与for-of结合的感觉。所以$.each对与json数据的解析也十分强大。

JavaScript中的数据属性与访问器属性

发表于 2017-07-09   |   1,151 字   |   4 min | 分类于 JavaScript

JavaScript中的对象在大多数OO的语言中都是非常容易理解的，即它就是一组散列的键值对。对它进行取值或赋值的操作都非常的简单明了。在看似简单的对象下，它还有些高级的特性。

属性

属性（property）是JavaScript对象中的key，在JS的内部，用特性（attribute）来属性的各种特征，而这些高级特性，是提供给JavaScript引擎所使用的，为了与属性区分开来，一般用[[Configurable]]这样两对方括号来表示它。

JavaScript对象的属性有两种类型，一是数据属性，另外一种是访问器属性。

数据属性

简介

在我们编写的JavaScript代码中涉及到的属性，几乎都是数据属性。

数据属性有四个特性用于描述它的特征：

  • [[Configurable]]: 用于表示此属性能否被删除（delte）从而重新定义，能够修改属性的其他特性，能否把属性改为访问器属性，默认值为true;
  • [[Enumerable]]： 表示能够通过for-in遍历出此属性，能够通过Object.keys()返回对象自身的所有可枚举的属性的键名，能否使用JSON.stringfy(),以及在ES6中能否使用Object.assign()拷贝对象，默认值为true；
  • [[Writable]]: 表示能否修改属性的值，默认值为true；
  • [[Value]]: 表示属性的值，默认值为undefined；

如果需要修改某个对象的属性的特性，需要使用Object.defineProperty这个方法，它有三个参数，第一个参数是你需要修改特性的所在的对象，第二个参数是需要修改哪一个属性，第三个参数是一个对象，即为上面四个特性的值，可以只传递一部分。

实践

var girl = {
name: 'LLXS',
age: 23,
job: 'F2E',
desc: function() {
    console.log(this.name)
    }
}

Object.defineProperty(girl,'name',{
    writable: false
})

girl.name = 'XYY'

console.log(girl.name); // LLXS

这里，通过Object.defineProperty将girl的name属性的writable特性设置成false，所以在后面即使手动修改了gril.name的值，最终在对象上并没有生效。在非严格模式下代码可以这样写，但是在严格模式下，将writable为false的属性重复赋值将会导致错误（如下图）

其他的特性也是通过这个方法来进行配置，这里就不一一赘述了。

应用

是浏览器环境下，undefined是window的一个属性，在浏览器中执行以下代码

window.undefined = 'define';
console.log(undefined); // undefined

那么，很明显undefined是一个writable为false的属性，通过getOwnPropertyDescriptor可以验证

var des = Object.getOwnPropertyDescriptor(window,undefined);
console.log(des);

从打印的结果中可以看到，window的undefined属性中value为默认的undeined，其余三个特性均为false，所以不能够对undefined重新赋值。

由此，我想到通过一下代码改变undefined的特性来是之可以被赋值

Object.defineProperty(window,undefined,{
    configurable: true,
    writable: true
})
window.undefined = 'define';
console.log(undefined)

结果

根据这里的提示，JavaScript为了安全，并不允许我们对undefined的特性进行重定义，所以就没有办法对undefined进行重新的赋值了。

访问器属性

简介

访问器属性不能直接定义，必须使用 Object.defineProperty() 来定义。它也有四个特性，其他两个与数据属性相同，如下：

  • [[Configurable]]: 用于表示此属性能否被删除（delte）从而重新定义，能够修改属性的其他特性，能否把属性改为访问器属性，默认值为true;
  • [[Enumerable]]： 表示能够通过for-in遍历出此属性，能够通过Object.keys()返回对象自身的所有可枚举的属性的键名，能否使用JSON.stringfy(),以及在ES6中能否使用Object.assign()拷贝对象，默认值为true；
  • [[Get]]: 在读取属性时调用的函数。默认值为undefined；
  • [[Set]]: 在写入属性时调用的函数。默认值为undefined；

实践

var book = {
    _year: 2004,
    edition: 1
};
Object.defineProperty(book, "year", {
    get: function () {
        return this._year;
    },
    set: function (newValue) {
        if (newValue > 2004) {
            this._year = newValue;
            this.edition += newValue - 2004;
        }
    }
});
book.year = 2005;
console.log(book.edition);//2

_year是只能通过对象属性访问的属性。而访问器属性year则包含一个getter函数和一个setter函数。getter函数返回_year的值，setter函数通过计算来确定正确的版本。这是使用访问器属性的常见方式，即设置一个属性的值会导致其他属性发生变化。需要注意的是，访问器属性是 ECMA 标准定义的属性类型，是用来实现 Javascript 引擎本身使用的，因此不建议开发者使用访问器属性。

众所周知BootStrap的自适应的，而且从小屏幕的手机到PC大屏幕都实现了自适应，单这肯定不是只有引用了bootstrap.css就可以自动实现的。开发者利用栅格式布局，通过对HTML添加适当的类，即可实现自适应。但是如上图展示的这张网页，引入bootstrap只是为了一个自适应的功能，未免太大材小用了，所以，利用bootstrap的这种栅格化的思想，我们自己实现了一个栅格式布局的样式库。

实现自己的栅格样式库

以下内容需要一丢丢的SASS与Compass的知识，如果你不会，那就花20分钟去看完这两篇文章。

首先新建一个_grid.scss，就在这个scss存放预留的自适应样式。首先，仿照bootstrap的栅格式布局，整个布局版面都是有行列撑起来的，一行里存在12列，列必须依附于行的父级存在。

行的代码比较简单，这里的两个margin不是必须的，然后将它的盒模型设置成IE的盒模型，因为它下边的列会涉及到浮动，所有最后引用compass清除一个浮动就可以了，代码如下。

.row {
  margin-right: -15px;
  margin-left: -15px;
  @include box-sizing(border-box);
  @include pie-clearfix;
}

列的代码量就非常的大了，以下之后涉及到一些关键的部分，且只考虑992px与1200px两种屏幕宽度尺寸，列命名为col-md-x和col-lg-x，两部分代码类似，只是在媒体查询部分不一样。栅格有12列，所以x即为1-12。

以大屏幕1200px为例，首先要让所以的列都向左浮动，然后十二分之一是8.33333333%，所以col-lg-x的width即为8.3333333% * x,代码如下：

  }

  }
  .col-lg-6 {
    width: 50%;
  }
  .col-lg-5 {
    width: 41.66666667%;
  }
  .col-lg-4 {
    width: 33.33333333%;
  }
  .col-lg-3 {
    width: 25%;
  }
  .col-lg-2 {
    width: 16.66666667%;
  }
  .col-lg-1 {

然后在小屏幕992px上的代码类似，只是需要把第一句改成@media (min-width: 992px)，后面的类名改为col-md-x。

代码都一样，但是也要根据不同的屏幕尺寸写个多份，这就是自适应的实现原理所在。要想根据不同的屏幕大小自适应布局，在引用的时候，就应该给标签套上多个类，根据不同的屏幕大小，显示不同的百分比。

<div class="row">
    <div class="col-md-12 col-lg-6">Hello</div>
    <div class="col-md-12 col-lg-6">World</div>
</div>

如上面这段最简单的自适应代码，在小屏幕（992px）上，生效的类是col-md-12，所以row内部的两个div各占width的100%，虽然它们均为左浮动，但依然如标准文档流一样上下排列。而在大屏幕（1200px）上，生效的则为col-lg-6这个类，width 50%，均左浮动，所以在这样的大屏幕上则是左右浮动的布局。

SASS结合Compass的安装与最基本使用

发表于 2017-06-18   |   821 字   |   3 min | 分类于 CSS

以前也了解过一点点的SASS、LESS方面的知识，但是基本限于只能定义个变量，写个嵌套这个程度，况且之前还不知有Compass这个逆天神器的存在。如今，在新同事的引导下，整个项目的页面完全是由Sass + Compass这种模式来开发的，这是一种完全不一样的开发模式，特别是Compass的存在，使得页面的开发进入了工程化、模块化的新阶段。

简介

限于笔者对SASS与Compass的理论掌握也只是皮毛，只是在工程中有过一点实际的应用所以才对其有更深入的理解，所以下面的阐述难免有些纰漏。

SASS不是一种新的语言，它只是书写CSS的一种新的方式，通过“编译器”，最终它还是会转化成CSS代码的，因为SASS并不能被HTML所解释。而这样一种新的方式，它的作用就是简化CSS的编写，同时提高整个系统的可维护性。

Compass则是一个工具与一系列的“库文件”，它与SASS之间的关系就好比jQuery与JavaScript的关系，利用Compass中提供的“库”，有许多的样式都不用我们自己编写，利用SASS的语法直接引用就可以了。

安装

前面提到了“编译器”的概念，而这里的“编译器”就是Ruby运行环境。

在Windows上安装Ruby，首先要下载RubyInstaller，这里注意选择Windows的位版本，整个安装过程都是傻瓜式的，也不需要手动配置环境变量。安装完成之后，在CMD里面ruby -v,如果显示有版本号，那么整个Ruby就在你的系统上安装成功了。

接下来要提到一个叫gem的东西，作为一个前端，可以不知道gem，但肯定知道npm，如果不知道什么是npm，建议您打开一个新的标签页Google一下，看十几分钟再回头来看这篇文章。在我的理解中gem就是Ruby下的npm。

在我们的Ruby环境安装好之后，gem也就自动安装好了，和npm类似，接下来就要配置其镜像地址gem sources -a ruby.taobao.org，这样就能在国内的网络下畅通的下个各个gem包。在SASS与Compas的使用上，我们需要的就是两个包，首先安装SASS，gem install sass，然后安装Compass，gem install compass，最后利用sass -v,compass -v分别测试一下SASS与Compass是否安装好，如果以前顺利，那么整个环境就已经建立成功了。

使用

进入你想要建立工程的目录，compass create myproject,即可建立一个Compass工程，在这里面，你可以看到一个名为config.rb的文件，里面配置了css、js、img等资源的输入输出配置，我们可以手动修改它。

在当前工程目录compass compile，即可编译整个工程的文件；compass watch，即可监听整个工程，在工程文件有变化时，可以立即改变整个工程内对应的文件。例如在sass目录下编写或者改变sass代码后，在css目录下对应的文件即会发生变化。

CSS部分

  1. 在开始书写CSS代码之前，这里要科普两个CSS3新属性，分别是background-clip和appearance。

    1. background-clip

      这个属性代表background所能覆盖的范围，我们都知道，默认情况下，background在盒模型中覆盖的区域是content + padding + border（IE是content + padding），所以就引出了background-clip这个属性的三个值——

      content-box:背景被裁剪到边框盒。

      padding-box:背景被裁剪到内边距框。

      border-box: 背景被裁剪到内容框。（默认）

    2. appearance

      这个属性在所有的浏览器中都不被支持，但是可以通过私有前缀在Firefox和Chrome中使用（-moz-appearance与-webkit-appearance）。它的作用是使元素看起来像标准的用户界面。

      normal 将元素呈现为常规元素。

      icon 将元素呈现为图标（小图片）。

      window 将元素呈现为视口。

      button 将元素呈现为按钮。

      menu 将元素呈现为一套供用户选择的选项。

      field 将元素呈现为输入字段。

      ps: -webkit-appearance: none /隐藏元素的默认显示/

  2. 自定义checkbox与radio样式的思想就是让元素默认的样式隐藏，然后通过border、background、定width/height的方式定义自己的样式，代码及详细注释如下

    /*清除outline，使其垂直top对齐*/
input {
    outline: none;
    vertical-align: top;
}

.radio,.checkbox {
    /*定宽高*/
    width: 1.2rem;
    height: 1.2rem;
    /*保留border-color，背景色仅覆盖到padding*/
    background-clip: padding-box;
    border: 1px solid grey;
    /*消除input的默认样式*/
    -webkit-appearance: none;
    /*使后面的“勾”相对其决定定位*/
    position: relative;
}

.radio {
    /*radio当然是圆的*/
    border-radius: 50%;
}

.radio:checked,.checkbox:checked {
    /*定义被选中时的背景色*/
    background-color: #04c9e8;
}

/*通过before伪元素定义“勾”的样式*/
.radio:checked:before,.checkbox:checked:before {
    content: "";
    /*四方居中*/
    position: absolute;
    left: 50%;
    top: 50%;
    margin-left: -0.25rem;
    margin-top: -0.25rem;

    border: 1px solid #fff;
    /*消除top与right方向的border，使其显示为“勾”*/
    border-top: none;
    border-right: none;

    width: 0.5rem;
    height: 0.3rem;

    /*旋转，使其显示为“勾”*/
    transform: rotate(-45deg);
}
