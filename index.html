栅格化布局的网页自适应

发表于 2017-06-25   |   1,509 字   |   6 min | 分类于 CSS

以前写网页几乎没有考虑到屏幕大小的问题，都是按照设计稿写死的布局，一般最常见的就是header & footer 100%，中间定宽的这种布局，只要不是屏幕太小或者过大，一般都不会出现什么问题，不过最近有项目不是这种布局，如果不针对不同的屏幕自适应的话，网页就会变得很丑，所以就借这个周末利用同事的(@liaojiajia)代码研究了一下，收获颇丰。

需求分析

然后，在width: 1920px的屏幕上是这样的

首先，在header与侧边栏上均没有任何变化，响应的地方都集中在了中间的content部分，为了方便表述，如1024px上呈现的图，将上面部分称为A，下面部分为B，A的三个item称为a1、a2、a3。

可以看到，在1024px的状态下，A、B上下排列，A的3个item浮动，宽度大约是父级的1/3；在1920px下，A与B左右浮动，A大概占父级的1/3,A下面的3个item换行对齐排列。

一直以来我都不太喜欢写响应式布局的代码，因为这需要在开发的时候考虑很多的情况，在根据不同的情况在写出不同的CSS，甚至有时候还需要改变HTML的代码。以前在学习响应式布局的时候，为了不考虑太多，就用一种非常智障的方式来写——在HTML中link样式的时候根据screen的width引入两套不同样式，所以就必须要写两套CSS，而且这两套样式中大部分代码也是重复的。

众所周知BootStrap的自适应的，而且从小屏幕的手机到PC大屏幕都实现了自适应，单这肯定不是只有引用了bootstrap.css就可以自动实现的。开发者利用栅格式布局，通过对HTML添加适当的类，即可实现自适应。但是如上图展示的这张网页，引入bootstrap只是为了一个自适应的功能，未免太大材小用了，所以，利用bootstrap的这种栅格化的思想，我们自己实现了一个栅格式布局的样式库。

实现自己的栅格样式库

以下内容需要一丢丢的SASS与Compass的知识，如果你不会，那就花20分钟去看完这两篇文章。

首先新建一个_grid.scss，就在这个scss存放预留的自适应样式。首先，仿照bootstrap的栅格式布局，整个布局版面都是有行列撑起来的，一行里存在12列，列必须依附于行的父级存在。

行的代码比较简单，这里的两个margin不是必须的，然后将它的盒模型设置成IE的盒模型，因为它下边的列会涉及到浮动，所有最后引用compass清除一个浮动就可以了，代码如下。

.row {
  margin-right: -15px;
  margin-left: -15px;
  @include box-sizing(border-box);
  @include pie-clearfix;
}

列的代码量就非常的大了，以下之后涉及到一些关键的部分，且只考虑992px与1200px两种屏幕宽度尺寸，列命名为col-md-x和col-lg-x，两部分代码类似，只是在媒体查询部分不一样。栅格有12列，所以x即为1-12。

以大屏幕1200px为例，首先要让所以的列都向左浮动，然后十二分之一是8.33333333%，所以col-lg-x的width即为8.3333333% * x,代码如下：

@media (min-width: 1200px) {
  .col-lg-1, .col-lg-2, .col-lg-3, .col-lg-4, .col-lg-5, .col-lg-6, .col-lg-7, .col-lg-8, .col-lg-9, .col-lg-10, .col-lg-11, .col-lg-12 {
    float: left;
  }
  .col-lg-12 {
    width: 100%;
  }
  .col-lg-11 {
    width: 91.66666667%;
  }
  .col-lg-10 {
    width: 83.33333333%;
  }
  .col-lg-9 {
    width: 75%;
  }
  .col-lg-8 {
    width: 66.66666667%;
  }
  .col-lg-7 {
    width: 58.33333333%;
  }
  .col-lg-6 {
    width: 50%;
  }
  .col-lg-5 {
    width: 41.66666667%;
  }
  .col-lg-4 {
    width: 33.33333333%;
  }
  .col-lg-3 {
    width: 25%;
  }
  .col-lg-2 {
    width: 16.66666667%;
  }
  .col-lg-1 {
    width: 8.33333333%;
  }
｝

然后在小屏幕992px上的代码类似，只是需要把第一句改成@media (min-width: 992px)，后面的类名改为col-md-x。

代码都一样，但是也要根据不同的屏幕尺寸写个多份，这就是自适应的实现原理所在。要想根据不同的屏幕大小自适应布局，在引用的时候，就应该给标签套上多个类，根据不同的屏幕大小，显示不同的百分比。

<div class="row">
    <div class="col-md-12 col-lg-6">Hello</div>
    <div class="col-md-12 col-lg-6">World</div>
</div>

如上面这段最简单的自适应代码，在小屏幕（992px）上，生效的类是col-md-12，所以row内部的两个div各占width的100%，虽然它们均为左浮动，但依然如标准文档流一样上下排列。而在大屏幕（1200px）上，生效的则为col-lg-6这个类，width 50%，均左浮动，所以在这样的大屏幕上则是左右浮动的布局。

实现上图中的效果

直接上代码

<div class="main">
    <div class="row">
        <div class="col-lg-4 col-md-12">
            <div class="row">
                <div class="col-lg-12 col-md-4"></div>
                <div class="col-lg-12 col-md-4"></div>
                <div class="col-lg-12 col-md-4"></div>
            </div>
        </div>
        <div class="col-lg-8 col-md-12"></div>
    </div>
</div>

在小屏幕小，A、B执行col-md-12,所以上下排列，A中的item，col-md-4,每个item各占1/3；在大屏幕下，A、B执行col-lg-4、col-lg-8,各占width的1/3、2/3，A中的item执行col-lg-12,所以均占父级的100%宽度。

总结

当然一个栅格化的代码库是不会这么简单的，这里只列举了其中最常用的一些代码，另外还有一些平移、定位之类功能，自己也正在研究。自适应是网页开发中的难点，但是随着多终端化的发展，这些技术也用的越来越多，在以后的项目中还会加深对这一块的学习。

SASS结合Compass的安装与最基本使用

发表于 2017-06-18   |   821 字   |   3 min | 分类于 CSS

以前也了解过一点点的SASS、LESS方面的知识，但是基本限于只能定义个变量，写个嵌套这个程度，况且之前还不知有Compass这个逆天神器的存在。如今，在新同事的引导下，整个项目的页面完全是由Sass + Compass这种模式来开发的，这是一种完全不一样的开发模式，特别是Compass的存在，使得页面的开发进入了工程化、模块化的新阶段。

简介

限于笔者对SASS与Compass的理论掌握也只是皮毛，只是在工程中有过一点实际的应用所以才对其有更深入的理解，所以下面的阐述难免有些纰漏。

SASS不是一种新的语言，它只是书写CSS的一种新的方式，通过“编译器”，最终它还是会转化成CSS代码的，因为SASS并不能被HTML所解释。而这样一种新的方式，它的作用就是简化CSS的编写，同时提高整个系统的可维护性。

Compass则是一个工具与一系列的“库文件”，它与SASS之间的关系就好比jQuery与JavaScript的关系，利用Compass中提供的“库”，有许多的样式都不用我们自己编写，利用SASS的语法直接引用就可以了。

安装

前面提到了“编译器”的概念，而这里的“编译器”就是Ruby运行环境。

在Windows上安装Ruby，首先要下载RubyInstaller，这里注意选择Windows的位版本，整个安装过程都是傻瓜式的，也不需要手动配置环境变量。安装完成之后，在CMD里面ruby -v,如果显示有版本号，那么整个Ruby就在你的系统上安装成功了。

接下来要提到一个叫gem的东西，作为一个前端，可以不知道gem，但肯定知道npm，如果不知道什么是npm，建议您打开一个新的标签页Google一下，看十几分钟再回头来看这篇文章。在我的理解中gem就是Ruby下的npm。

在我们的Ruby环境安装好之后，gem也就自动安装好了，和npm类似，接下来就要配置其镜像地址gem sources -a ruby.taobao.org，这样就能在国内的网络下畅通的下个各个gem包。在SASS与Compas的使用上，我们需要的就是两个包，首先安装SASS，gem install sass，然后安装Compass，gem install compass，最后利用sass -v,compass -v分别测试一下SASS与Compass是否安装好，如果以前顺利，那么整个环境就已经建立成功了。

使用

进入你想要建立工程的目录，compass create myproject,即可建立一个Compass工程，在这里面，你可以看到一个名为config.rb的文件，里面配置了css、js、img等资源的输入输出配置，我们可以手动修改它。

在当前工程目录compass compile，即可编译整个工程的文件；compass watch，即可监听整个工程，在工程文件有变化时，可以立即改变整个工程内对应的文件。例如在sass目录下编写或者改变sass代码后，在css目录下对应的文件即会发生变化。

纯CSS美化单选框与复选框

发表于 2017-05-28   |   748 字   |   3 min | 分类于 CSS

之前在开发的过程中也遇到过改变单选框与复选框样式的问题，但就是自己懒，忙于赶进度，这些方面就没有进一步的去优化它。结果不久前面试的时候被问到这个问题，果不其然没有回答上来。所以刚才研究了一下，写了一个小栗子来实现这个功能。

需求分析

原生的radio和checkbox长这样

样式美化后的radio和checkbox长成了这样

HTML部分

这里没什么好说的，直接贴代码

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <title>Document</title>
    <link rel="stylesheet" type="text/css" href="css/style.css">
</head>
<body>
    <label for="cd"><input id="cd" type="radio" name="city" class="radio">成都</label>
    <label for="hz"><input id="hz" type="radio" name="city" class="radio">杭州</label>
    <hr>
    <label for="asus"><input type="checkbox" class="checkbox" id="asus">华硕</label>
    <label for="app"><input type="checkbox" class="checkbox" id="app">苹果</label>
    <label for="hw"><input type="checkbox" class="checkbox" id="hw">华为</label>
</body>
</html>

CSS部分

  1. 在开始书写CSS代码之前，这里要科普两个CSS3新属性，分别是background-clip和appearance。

    1. background-clip

      这个属性代表background所能覆盖的范围，我们都知道，默认情况下，background在盒模型中覆盖的区域是content + padding + border（IE是content + padding），所以就引出了background-clip这个属性的三个值——

      content-box:背景被裁剪到边框盒。

      padding-box:背景被裁剪到内边距框。

      border-box: 背景被裁剪到内容框。（默认）

    2. appearance

      这个属性在所有的浏览器中都不被支持，但是可以通过私有前缀在Firefox和Chrome中使用（-moz-appearance与-webkit-appearance）。它的作用是使元素看起来像标准的用户界面。

      normal 将元素呈现为常规元素。

      icon 将元素呈现为图标（小图片）。

      window 将元素呈现为视口。

      button 将元素呈现为按钮。

      menu 将元素呈现为一套供用户选择的选项。

      field 将元素呈现为输入字段。

      ps: -webkit-appearance: none /隐藏元素的默认显示/

  2. 自定义checkbox与radio样式的思想就是让元素默认的样式隐藏，然后通过border、background、定width/height的方式定义自己的样式，代码及详细注释如下

    /*清除outline，使其垂直top对齐*/
input {
    outline: none;
    vertical-align: top;
}

.radio,.checkbox {
    /*定宽高*/
    width: 1.2rem;
    height: 1.2rem;
    /*保留border-color，背景色仅覆盖到padding*/
    background-clip: padding-box;
    border: 1px solid grey;
    /*消除input的默认样式*/
    -webkit-appearance: none;
    /*使后面的“勾”相对其决定定位*/
    position: relative;
}

.radio {
    /*radio当然是圆的*/
    border-radius: 50%;
}

.radio:checked,.checkbox:checked {
    /*定义被选中时的背景色*/
    background-color: #04c9e8;
}

/*通过before伪元素定义“勾”的样式*/
.radio:checked:before,.checkbox:checked:before {
    content: "";
    /*四方居中*/
    position: absolute;
    left: 50%;
    top: 50%;
    margin-left: -0.25rem;
    margin-top: -0.25rem;

    border: 1px solid #fff;
    /*消除top与right方向的border，使其显示为“勾”*/
    border-top: none;
    border-right: none;

    width: 0.5rem;
    height: 0.3rem;

    /*旋转，使其显示为“勾”*/
    transform: rotate(-45deg);
}

webpack之安装与命令行

发表于 2017-05-28   |   659 字   |   3 min | 分类于 框架

webpack简介

webpack是一个代码打包器，依附于NPM平台。在前端领域，所谓的代码打包，即是将所有模块化后的代码打包到一个JS文件中，HTML中只需引入这个打包后的文件即可，实际在浏览器中运行的也是这一个文件。

除了对JS文件进行打包之外，还可以对其他的非JS文件进行打包，如打包后的CSS引入JS文件中。

当然除了打包这一功能之外，webpack还有其他的功能。在模块化上，它支持CommonJS、AMD、CMD、ES6多种模块化方式，并且可以在一个功能中混用。依附于webpack同样有相当多的插件，用来实现网页的热部署，利用babel来讲ES6转ES5等。

接下来，将介绍webpack的简单使用与相关命令。

webpack代码的编写

新建工程webpack-test

利用npm初始化整个工程

在CMD中

npm init // 命令行中与填写的字段全部Enter跳过

之后，即可在工程根目录下生成package.json文件

安装webpack

在CMD中

npm install webpack --save-dev

安装完成之后，与webpack相关的依赖将会存放在工程根目录下的node_modules文件夹中

安装完成

至此，在当前工程下的webpack环境就安装完成。

webpack命令

webpack打包

  1. 在根目录下新建文件hello.js

    import './world.js'

function hello(str) {
    alert(str);
}

hello('Test Webpack')

  2. 在根目录下新建文件world.js

    function world() {
    return {

    }
}

  3. 在CMD中执行命令

    webpack hello.js hello.bundle.js

    执行上述操作后，即可将hello.js和与之关联的文件进行打包，打包文件为hello.bundle.js，这个bundle文件存放在工程的根目录下；

  4. 通过HTML文件对之进行引用，新建index.html,对hello.bundle.js进行引用

    <!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <title>Demo</title>
</head>
<body>
    <h1>webpack</h1>
    <script type="text/javascript" src="hello.bundle.js"></script>
</body>
</html>

打开网页后，在浏览器中就会有一个“Test Webpack”的alert。

webpack打包CSS

  1. 新建样式文件style.css

    body {
    background: red;
}

  2. 引入需要的依赖

    // 在CMD中
    npm install style-loader css-loader

  3. 引入CSS文件

    1. 在hello.js中引入

      import 'style-loader!css-loader!./style.css';

    2. CMD中加载

      1. hello.js

        import './style.css';

      2. CMD命令中

        webpack hello.js hello.bundle.js --module-bind "css=style-loader!css-loader"

热部署

webpack hello.js hello.bundle.js --watch

观察打包过程

webpack hello.js hello.bundle.js --progess

列出所有打包的模块(包括处理方式)

webpack hello.js hello.bundle.js --display-modules

加载模块的原因

webpack hello.js hello.bundle.js  --display-reasons
