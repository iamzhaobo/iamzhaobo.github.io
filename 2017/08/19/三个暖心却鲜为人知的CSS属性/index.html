三个暖心却鲜为人知的CSS属性

发表于 2017-08-19   |   909 字   |   4 min | 分类于 CSS |

平时工作常用到的CSS属性就那几十个，在编写代码中用到的也不是特别深的东西，不过对于一些生僻的属性，使用了之后可以大大的方便我们的开发工作。

pointer-events

在不考虑SVG元素的情况下，这个属性对普通的元素有两个值：auto(默认值)和none；在将值设置为none的情况下，有两个含义，一是此元素不能不能捕获任何的鼠标事件，而是鼠标会完全忽略测元素，穿够作用于下一层元素。

看结构代码：

<body>
    <div class="wrap">
        <select>
            <option value="">AAA</option>
            <option value="">BBB</option>
            <option value="">CCC</option>
        </select>
        <canvas id="c" width="500" height="400">什么杰宝浏览器不支持Canvas</canvas>
    </div>

    <div class="btn-wrap">
        <button type="button">BUTTON</button>
    </div>

    <script>
        var canvas = document.querySelector('#c');
        var context = canvas.getContext('2d');
        context.fillStyle = 'rgba(250,0,0,.5)';
        context.fillRect(0,0,500,400);

        var btnWrap = document.querySelector('.btn-wrap');
        btnWrap.addEventListener('click',function() {
            alert('Btn Wrap');
        })

        var btn = document.querySelector('button');
        btn.addEventListener('click',function(event) {
            alert('Fuck')
        })
    </script>
</body>

浏览器显示如下：

CSS代码如下：

.wrap {
    width: 500px;
    height: 400px;
    border: 1px solid #ccc;
}

select {
    position: absolute;
}

#c {
    position: absolute;
    pointer-events: none;
}

.btn-wrap {
    border: 1px solid yellow;
    pointer-events: none;
}

button {
    width: 100px;
    height: 30px;
    margin-top: 10px;
    pointer-events: auto;
}

按理说，在这个0.5透明度的红色Canvas的作用下，里面的select应该是不能点击的，但是由于给这个canvas设置了pointer-events: none,鼠标直接穿透了canvas点击到了Select；

在下边.btn-wrap的地方，已经将btn-wrap设置成了pointer-events: none，所以直接点yellow色内部的区域是不能处罚点击事件的，但是，这个CSS属性并不能阻止冒泡，在点击了button之后，执行了btn绑定的事件代码，就会冒泡到外层直接.btn-wrap绑定的事件代码。

看具体的代码实例请狠狠的点击这里

::selection

selection是一个伪元素（注意不是伪元素），当我们在浏览器中选中文字(以Chrome浏览器为例)，呈现出来的效果是淡蓝底白色文字的，通过配置selection伪元素即可改变文字的样式。
看以下代码： 

<html lang="en">
    <head>
        <meta charset="UTF-8">
        <title>Demo</title>
        <style>
            p::selection {
                color: red;
                background: blue;
                text-decoration: underline;
            }
            p::-moz-selection {
                color: red;
                background: blue;
                text-decoration: underline;
            }
        </style>
    </head>
    <body>
        <p>我要选中这段文字啦啦啦啦</p>
        <div>我要选中这段普通的文字啦啦啦啦</div>
    </body>
    </html>

要看效果请点击这里

需要注意的是不是什么属性都可以网selection这个伪元素里边写，目前它只支持一小部分CSS属性：color, background-color, cursor, outline, text-decoration, text-emphasis-color和text-shadow。

user-select

这一个属性也是与选中文本相关的，在网上有些资料为了保护版权，有些就会采用这样一种(比较low)的方法，它有四个属性值none、text、all、element；

text是它的默认值，表示可以选择文本；
none则是需要用到这个属性时最为常用的，表示不能选中文本；
all是指能够在单击的时候一并选中当前文字，就好似我们双击选中一个英文单词一样；
element表示可以选择文本，但选择范围受元素边界的约束；

这里写了一个小demo，使用了除默认值之外的三个特性，都非常的简单，但是需要主要的是chrome浏览器目前并不支持element这个值，而且经在支持这个属性的浏览器(IE10+,Firefox4+)上的表现与text并无不同。还没有搞清楚这是什么原因。
