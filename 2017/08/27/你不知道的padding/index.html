你不知道的padding

发表于 2017-08-27   |   1,031 字   |   4 min | 分类于 CSS |

最近在二刷张鑫旭老师的《深入理解CSS系列》，以下内容均是课程笔记与实践，更多详细的内容请看这里.

for block

padding是盒模型的重要组成部分，对于块级元素来说，又可以分为两类。一是W3C标准盒模型，即在box-sizing: content-box的情况下，padding的增加后会导致整个元素尺寸的增大；二是IE盒模型，即在box-sizing: border-box的情况下，padding的增加不会导致元素尺寸的增加，增加的padding会导致content区域减少。

如上是知乎的主页，利用这一特性，在对盒子进行绘制的时候，一般可以采用IE的盒模型，然后利用padding将四周的留白写出来。

虽然在border-box的情况下，padding不会对块级元素的尺寸造成影响，但是当水平或垂直方向上的padding之和大于width或height之后，盒子就会“爆炸”了，如这里的例子，当发生了这种情况时，content区域将会变成0，width或height将只有padding决定。

for inline

在inline下，padding有一个“奇怪”的特征。

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <title>Demo</title>
    <style>
        span {
            background: red;
            padding: 1px 2px;
        }
    </style>
</head>
<body>
    <div>我在上面我在上面我在上面</div>
    <div>我在左边<span>MIDDLE</span>我在右边</div>
    <div>我在下面我在下面我在下面</div>
</body>
</html>

其他特性

首先padding不支持负值，它支持百分比，百分比是按照元素的width来计算的，利用padding支持百分比这个属性，我们可以轻易的在一个块级元素下画出一个正方向。

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <title>Demo</title>
    <style>
        .outer {
            width: 200px;
            height: 500px;
            border: 3px dotted red;
        }

        .inner {
            padding: 50%;
            background: cyan;
        }
    </style>
</head>
<body>
    <div class="outer">
        <div class="inner"></div>
    </div>
</body>
</html>

原理非常的简单，这里就不解释了。它的应用场景在什么地方呢？移动端！

如类型微信朋友圈顶部的图片，假如说它是一个正方形，那么利用这一特性，用body为“正方形”的父级，那么我们就可以在任意屏幕的手机上，画出一个正方形。

另外在inline元素的padding会让strut元素出现，’strut’本质上文本的content area区域的体现，所以可以通过font-size:0让其消失。

比如下面这个非主流的“登录|注册”写法。

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <title>Demo</title>
    <style>
        span {
                font-size: 0;
                padding: 11px 6px 2px;
                border-left: 2px solid;
                margin-left: 10px;
        }
    </style>
</head>
<body>
    <div>登录<span></span>注册</div>
</body>
</html>

当我把font-size:0去掉之后

大家都知道在inline-block下font-size != 0 时会有4px的间距，但是在inline下依然有“不正常”的情况出现，所以在使用这两种元素的时候如果遇到莫名其妙的问题，最好能够font-size:0试一试。

小技巧

汉堡菜单

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <title>Demo</title>
    <style type="text/css">
        div {
            width: 40px;
            height: 5px;
            background: red;
            padding: 10px 0;
            background-clip: content-box;
            border-top: 5px solid red;
            border-bottom: 5px solid red;
        }
    </style>
</head>
<body>
    <div></div>
</body>
</html>

完整代码这里

圆环

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <title>Demo</title>
    <style type="text/css">
        div {
            width: 50px;
            height: 50px;
            background: blue;
            border-radius: 50%;
            border: 3px solid blue;
            padding: 5px;
            background-clip: content-box;
        }
    </style>
</head>
<body>
    <div></div>
</body>
</html>

完整代码这里

其实它们的原理都是一样的，利用background-clip:content-box;让其只显示content-area部分，然后padding留出间隙，最外层用border填充。
