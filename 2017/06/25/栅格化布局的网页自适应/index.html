栅格化布局的网页自适应

发表于 2017-06-25   |   1,509 字   |   6 min | 分类于 CSS |

以前写网页几乎没有考虑到屏幕大小的问题，都是按照设计稿写死的布局，一般最常见的就是header & footer 100%，中间定宽的这种布局，只要不是屏幕太小或者过大，一般都不会出现什么问题，不过最近有项目不是这种布局，如果不针对不同的屏幕自适应的话，网页就会变得很丑，所以就借这个周末利用同事的(@liaojiajia)代码研究了一下，收获颇丰。

首先，在header与侧边栏上均没有任何变化，响应的地方都集中在了中间的content部分，为了方便表述，如1024px上呈现的图，将上面部分称为A，下面部分为B，A的三个item称为a1、a2、a3。

可以看到，在1024px的状态下，A、B上下排列，A的3个item浮动，宽度大约是父级的1/3；在1920px下，A与B左右浮动，A大概占父级的1/3,A下面的3个item换行对齐排列。

一直以来我都不太喜欢写响应式布局的代码，因为这需要在开发的时候考虑很多的情况，在根据不同的情况在写出不同的CSS，甚至有时候还需要改变HTML的代码。以前在学习响应式布局的时候，为了不考虑太多，就用一种非常智障的方式来写——在HTML中link样式的时候根据screen的width引入两套不同样式，所以就必须要写两套CSS，而且这两套样式中大部分代码也是重复的。

众所周知BootStrap的自适应的，而且从小屏幕的手机到PC大屏幕都实现了自适应，单这肯定不是只有引用了bootstrap.css就可以自动实现的。开发者利用栅格式布局，通过对HTML添加适当的类，即可实现自适应。但是如上图展示的这张网页，引入bootstrap只是为了一个自适应的功能，未免太大材小用了，所以，利用bootstrap的这种栅格化的思想，我们自己实现了一个栅格式布局的样式库。

实现自己的栅格样式库

以下内容需要一丢丢的SASS与Compass的知识，如果你不会，那就花20分钟去看完这两篇文章。

首先新建一个_grid.scss，就在这个scss存放预留的自适应样式。首先，仿照bootstrap的栅格式布局，整个布局版面都是有行列撑起来的，一行里存在12列，列必须依附于行的父级存在。

行的代码比较简单，这里的两个margin不是必须的，然后将它的盒模型设置成IE的盒模型，因为它下边的列会涉及到浮动，所有最后引用compass清除一个浮动就可以了，代码如下。

.row {
  margin-right: -15px;
  margin-left: -15px;
  @include box-sizing(border-box);
  @include pie-clearfix;
}

列的代码量就非常的大了，以下之后涉及到一些关键的部分，且只考虑992px与1200px两种屏幕宽度尺寸，列命名为col-md-x和col-lg-x，两部分代码类似，只是在媒体查询部分不一样。栅格有12列，所以x即为1-12。

以大屏幕1200px为例，首先要让所以的列都向左浮动，然后十二分之一是8.33333333%，所以col-lg-x的width即为8.3333333% * x,代码如下：

@media (min-width: 1200px) {
  .col-lg-1, .col-lg-2, .col-lg-3, .col-lg-4, .col-lg-5, .col-lg-6, .col-lg-7, .col-lg-8, .col-lg-9, .col-lg-10, .col-lg-11, .col-lg-12 {
    float: left;
  }
  .col-lg-12 {
    width: 100%;
  }
  .col-lg-11 {
    width: 91.66666667%;
  }
  .col-lg-10 {
    width: 83.33333333%;
  }
  .col-lg-9 {
    width: 75%;
  }
  .col-lg-8 {
    width: 66.66666667%;
  }
  .col-lg-7 {
    width: 58.33333333%;
  }
  .col-lg-6 {
    width: 50%;
  }
  .col-lg-5 {
    width: 41.66666667%;
  }
  .col-lg-4 {
    width: 33.33333333%;
  }
  .col-lg-3 {
    width: 25%;
  }
  .col-lg-2 {
    width: 16.66666667%;
  }
  .col-lg-1 {
    width: 8.33333333%;
  }
｝

然后在小屏幕992px上的代码类似，只是需要把第一句改成@media (min-width: 992px)，后面的类名改为col-md-x。

代码都一样，但是也要根据不同的屏幕尺寸写个多份，这就是自适应的实现原理所在。要想根据不同的屏幕大小自适应布局，在引用的时候，就应该给标签套上多个类，根据不同的屏幕大小，显示不同的百分比。

<div class="row">
    <div class="col-md-12 col-lg-6">Hello</div>
    <div class="col-md-12 col-lg-6">World</div>
</div>

如上面这段最简单的自适应代码，在小屏幕（992px）上，生效的类是col-md-12，所以row内部的两个div各占width的100%，虽然它们均为左浮动，但依然如标准文档流一样上下排列。而在大屏幕（1200px）上，生效的则为col-lg-6这个类，width 50%，均左浮动，所以在这样的大屏幕上则是左右浮动的布局。

实现上图中的效果

直接上代码

<div class="main">
    <div class="row">
        <div class="col-lg-4 col-md-12">
            <div class="row">
                <div class="col-lg-12 col-md-4"></div>
                <div class="col-lg-12 col-md-4"></div>
                <div class="col-lg-12 col-md-4"></div>
            </div>
        </div>
        <div class="col-lg-8 col-md-12"></div>
    </div>
</div>

在小屏幕小，A、B执行col-md-12,所以上下排列，A中的item，col-md-4,每个item各占1/3；在大屏幕下，A、B执行col-lg-4、col-lg-8,各占width的1/3、2/3，A中的item执行col-lg-12,所以均占父级的100%宽度。

总结

当然一个栅格化的代码库是不会这么简单的，这里只列举了其中最常用的一些代码，另外还有一些平移、定位之类功能，自己也正在研究。自适应是网页开发中的难点，但是随着多终端化的发展，这些技术也用的越来越多，在以后的项目中还会加深对这一块的学习。
